BarQC parameters
usage: BarQC [-h] -ss File -id INPUT_DIR -od OUT_DIR -gr GROUP [-su SUFFIX]
                [-pr PROCESSES] [-lo LOG_FILE] [-ea EMAIL_ADDRESS] [-v]

Description:
Perform a quality control on raw sequence data to estimate sequence performance.
For every fq/FASTQ file MD5sum, sequence lane and total reads are calculated
and outputted to given sample sheet.

optional arguments:
  -h, --help            show this help message and exit
  -su SUFFIX, --suffix SUFFIX
                        Define the suffix of read files. (default: .fq.gz)
  -pr PROCESSES, --processes PROCESSES
                        Define the amount of parallel processes. (default: 8)
  -lo LOG_FILE, --log_file LOG_FILE
                        Indicate the file path of the log file. (default:
                        BarQC.log)
  -ea EMAIL_ADDRESS, --email_address EMAIL_ADDRESS
                        Indicate personal email address if you want to be
                        notified when BarQC is finished. (default: None)

Required arguments:
  -ss File, --sample_sheet File
                        Define the reference file. (default: None)
  -id INPUT_DIR, --input_dir INPUT_DIR
                        Define the directory containing the raw sequencing
                        files. (default: None)
  -od OUT_DIR, --out_dir OUT_DIR
                        Define the output directory for statistics. (default:
                        None)
  -gr GROUP, --group GROUP
                        Indicate the group ID. (default: None)
BarMapping parameters
usage: BarMapping [-h] -re REF_FILE -id INPUT_DIR -od OUT_DIR -gr GROUP
                     [-su SUFFIX] [-qf QUALITY_FILTER] [-lo LOG_FILE]
                     [-bf BED_FILE] [-sd STATS_OUT_DIR] [-pr PROCESSES]
                     [-ea EMAIL_ADDRESS] [-v]

Description:
Map fq/FASTQ read data to a given reference file using BWA MEM in parallel.
For every read file, a BAM file, Q20 BAM file and Q20 Read Group BAM file are generated.
If applied statistics and plots are generated in the given output directory.

optional arguments:
  -h, --help            show this help message and exit
  -su SUFFIX, --suffix SUFFIX
                        Define the suffix of read files. (default: .fq.gz)
  -qf QUALITY_FILTER, --quality_filter QUALITY_FILTER
                        Indicate the quality filtering used for SamTools.
                        (default: 20)
  -lo LOG_FILE, --log_file LOG_FILE
                        Indicate the file path of the log file. (default:
                        BarMapping.log)
  -bf BED_FILE, --bed_file BED_FILE
                        Define the file path to the BED file of pre-defined
                        regions. (default: None)
  -sd STATS_OUT_DIR, --stats_out_dir STATS_OUT_DIR
                        Define the output directory for mapping statistics or
                        plots. (default: None)
  -pr PROCESSES, --processes PROCESSES
                        Define the amount of parallel processes. (default: 8)
  -ea EMAIL_ADDRESS, --email_address EMAIL_ADDRESS
                        Indicate personal email address if you want to be
                        notified when BarMapping is finished. (default: None)

Required arguments:
  -re REF_FILE, --ref_file REF_FILE
                        Define the reference file. (default: None)
  -id INPUT_DIR, --input_dir INPUT_DIR
                        Define the directory containing read fq files.
                        (default: None)
  -od OUT_DIR, --out_dir OUT_DIR
                        Define the output directory for BAM files. (default:
                        None)
  -gr GROUP, --group GROUP
                        Indicate the group ID. (default: None)
BarGenotype parameters
usage: BarGenotype [-h] -re REF_FILE -id INPUT_DIR -od OUT_DIR -gr GROUP
                      -pd PLOIDY -bf BED_FILE [-su SUFFIX] [-m MEMORY]
                      [-p PROCESSES] [-bs BATCH_SIZE] [-lo LOG_FILE] [-ba]
                      [-ea EMAIL_ADDRESS] [-v]

Description:
Genotype BAM files using GATK HaplotypeCaller in parallel.
After genotyping GenomicsDB is used to add batches of file to a single DB.
Finally a consensus VCF file is generated of all found SNPs across the samples.
This file can be filtered using BarVCF.

optional arguments:
  -h, --help            show this help message and exit
  -su SUFFIX, --suffix SUFFIX
                        Define the suffix of read files. (default: .bam)
  -m MEMORY, --memory MEMORY
                        Define the amount of memory in GB. (default: 25)
  -p PROCESSES, --processes PROCESSES
                        Define the number of parallel processes. (default: 8)
  -bs BATCH_SIZE, --batch_size BATCH_SIZE
                        Define the batch size used for genomics DB. (default:
                        100)
  -lo LOG_FILE, --log_file LOG_FILE
                        Indicate the file path of the log file. (default:
                        BarGenotype.log)
  -ba, --bamout         If applied, realigned BAM files will be saved
                        (default: False)
  -ea EMAIL_ADDRESS, --email_address EMAIL_ADDRESS
                        Indicate personal email address if you want to be
                        notified when BarGenotype is finished. (default: None)

Required arguments:
  -re REF_FILE, --ref_file REF_FILE
                        Define the reference file. (default: None)
  -id INPUT_DIR, --input_dir INPUT_DIR
                        Define the directory containing BAM files. (default:
                        None)
  -od OUT_DIR, --out_dir OUT_DIR
                        Define the output directory for BAM files. (default:
                        None)
  -gr GROUP, --group GROUP
                        Indicate the group ID. (default: None)
  -pd PLOIDY, --ploidy PLOIDY
                        Indicate the ploidy for the group. (default: None)
  -bf BED_FILE, --bed_file BED_FILE
                        Input BED file used for SNP calling. (default: None)
BarHaplotype parameters
usage: BarHaplotype [-h] -vf VCF_FILE -id INPUT_DIR -od OUT_DIR -gr GROUP
                       -pd PLOIDY -bf BED_FILE -re REF_FILE [-su SUFFIX]
                       [-p PROCESSES] [-ml MIN_READ_LENGTH] [-lo LOG_FILE]
                       [-ea EMAIL_ADDRESS] [-v]

Description:
Phase SNP variants for a given VCF file and .BAM files using WhatsHap in parallel.
All BAM files are down samples, to increase phasing performance.
Phasing is performed in batches and merged in to a single VCF file.
After phasing is performed, haplotypes are generated and outputted in the given output format.

optional arguments:
  -h, --help            show this help message and exit
  -su SUFFIX, --suffix SUFFIX
                        Define the suffix of read files. (default: .BWA.bam)
  -p PROCESSES, --processes PROCESSES
                        Define the number of parallel processes. (default: 8)
  -ml MIN_READ_LENGTH, --min_read_length MIN_READ_LENGTH
                        Define the minimum read length used for down sampling.
                        (default: 75)
  -lo LOG_FILE, --log_file LOG_FILE
                        Indicate the file path of the log file. (default:
                        BarHaplotype.log)
  -ea EMAIL_ADDRESS, --email_address EMAIL_ADDRESS
                        Indicate personal email address if you want to be
                        notified when BarHaplotype is finished. (default:
                        None)

Required arguments:
  -vf VCF_FILE, --vcf_file VCF_FILE
                        Define the VCF file to be phased. (default: None)
  -id INPUT_DIR, --input_dir INPUT_DIR
                        Define the directory containing BAM files. (default:
                        None)
  -od OUT_DIR, --out_dir OUT_DIR
                        Define the output directory for BAM files. (default:
                        None)
  -gr GROUP, --group GROUP
                        Indicate the group ID. (default: None)
  -pd PLOIDY, --ploidy PLOIDY
                        Indicate the ploidy for the group. (default: None)
  -bf BED_FILE, --bed_file BED_FILE
                        Input BED file used for down sampling the reads.
                        (default: None)
  -re REF_FILE, --ref_file REF_FILE
                        Input reference file used for phasing. (default: None)
BarVCF parameters
usage: BarVCF [-h] -vf VCF_FILE -od OUT_DIR -gr GROUP -pd PLOIDY -bf
                 BED_FILE [-nf] [-mia MIN_ALLELE_FREQ] [-maa MAX_ALLELE_FREQ]
                 [-mf MIN_SNP_FREQ] [-mq MIN_QUALITY] [-pf PEDIGREE_FILE]
                 [-mp MAX_PEDIGREE_SNP_ERROR] [-lo LOG_FILE]
                 [-ea EMAIL_ADDRESS] [-v]

Description:
Analyse a generated VCF file and create SNP output for Cervus or Genomic Selection.
If a pedigree file is given, SNPs are filtered according a ratio of a marker being correct
given the offspring, mother & father sample.

optional arguments:
  -h, --help            show this help message and exit
  -nf, --no_filter      If applied, filtering step of SNPs is skipped
                        completely. (default: False)
  -mia MIN_ALLELE_FREQ, --min_allele_freq MIN_ALLELE_FREQ
                        Define the minimum allele frequency for SNP filtering.
                        (default: 0.001)
  -maa MAX_ALLELE_FREQ, --max_allele_freq MAX_ALLELE_FREQ
                        Define the maximum allele frequency for SNP filtering.
                        (default: 0.85)
  -mf MIN_SNP_FREQ, --min_snp_freq MIN_SNP_FREQ
                        Define the minimum frequency for a called SNP being
                        present across all samples. (default: 0.95)
  -mq MIN_QUALITY, --min_quality MIN_QUALITY
                        Define the minimum SNP quality for SNP filtering.
                        (default: 250)
  -pf PEDIGREE_FILE, --pedigree_file PEDIGREE_FILE
                        If applied, SNPs are filtered according to the known
                        paircross samples. (default: None)
  -mp MAX_PEDIGREE_SNP_ERROR, --max_pedigree_snp_error MAX_PEDIGREE_SNP_ERROR
                        Define the maximum error frequency for a SNP across
                        the paircross samples. Only used when pedigree file is
                        given. (default: 0.1)
  -lo LOG_FILE, --log_file LOG_FILE
                        Indicate the file path of the log file. (default:
                        BarVCF.log)
  -ea EMAIL_ADDRESS, --email_address EMAIL_ADDRESS
                        Indicate personal email address if you want to be
                        notified when BarVCF is finished. (default: None)

Required arguments:
  -vf VCF_FILE, --vcf_file VCF_FILE
                        Define the VCF file to be analysed. (default: None)
  -od OUT_DIR, --out_dir OUT_DIR
                        Define the output directory for statistics. (default:
                        None)
  -gr GROUP, --group GROUP
                        Indicate the group ID. (default: None)
  -pd PLOIDY, --ploidy PLOIDY
                        Indicate the ploidy for the group and input VCF file.
                        (default: None)
  -bf BED_FILE, --bed_file BED_FILE
                        Input BED file used for down sampling the reads.
                        (default: None)
BarPaternity parameters
usage: BarPaternity [-h] -mf MARKER_FILE -of OFFSPRING_FILE -cf
                       CANDIDATE_FATHERS -od OUT_DIR -gr GROUP
                       [-gf GENOTYPE_FILE] [-cd CRITICAL_DELTA]
                       [-pl PAIR_LOD_SCORE] [-td TRIO_DELTA_SCORE]
                       [-lo LOG_FILE] [-ea EMAIL_ADDRESS] [-v]

Description:
Perform a paternity analysis on a generated marker file by BarVCF.
Paternity analysis is performed by Cervus, on the input offspring and candidate fathers file.
Generated output is transformed and saved in the output directory.
If a genotype file is applied, genotypes are added to output file.
In addition, a maternal check is applied to detect selfings or wrong sampled mothers.

optional arguments:
  -h, --help            show this help message and exit
  -gf GENOTYPE_FILE, --genotype_file GENOTYPE_FILE
                        Define the file path to the genotype file. See
                        Examples/Genotype_file.txt (default: None)
  -cd CRITICAL_DELTA, --critical_delta CRITICAL_DELTA
                        Define the critical delta score for Cervus. (default:
                        95)
  -pl PAIR_LOD_SCORE, --pair_lod_score PAIR_LOD_SCORE
                        Define the minimum Pair LOD score for given mother and
                        resulting father. (default: 0)
  -td TRIO_DELTA_SCORE, --trio_delta_score TRIO_DELTA_SCORE
                        Define the minimum Trio DELTA score for offspring,
                        given mother and resulting father. (default: 0)
  -lo LOG_FILE, --log_file LOG_FILE
                        Indicate the file path of the log file. (default:
                        BarPaternity.log)
  -ea EMAIL_ADDRESS, --email_address EMAIL_ADDRESS
                        Indicate personal email address if you want to be
                        notified when BarPaternity is finished. (default:
                        None)

Required arguments:
  -mf MARKER_FILE, --marker_file MARKER_FILE
                        Define the marker file generated by BarVCF. See
                        Examples/Cervus_marker_file.txt (default: None)
  -of OFFSPRING_FILE, --offspring_file OFFSPRING_FILE
                        Define the offspring with corresponding Mother sample
                        file.See Examples/Offspring.txt (default: None)
  -cf CANDIDATE_FATHERS, --candidate_fathers CANDIDATE_FATHERS
                        Define the file with candidate fathers.See
                        Examples/Candidate_fathers.txt (default: None)
  -od OUT_DIR, --out_dir OUT_DIR
                        Define the output directory for statistics. (default:
                        None)
  -gr GROUP, --group GROUP
                        Indicate the group ID. (default: None)
